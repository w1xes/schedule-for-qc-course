## Функціонал
Модуль розміщення пари на конкретний час/день у розкладі через адмін-панель. Процес охоплює кроки від вибору групи до остаточного підтвердження аудиторії та перевірки конфліктів.

## Обрана техніка
State Transition Testing (Тестування переходів між станами) + елементи Decision Table для перевірки конфліктів.

## Обґрунтування
Процес додавання пари є послідовним багатоекранним/багатокроковим флоу. Користувач не може обрати аудиторію, не обравши попередньо групу та часовий слот. Крім того, система має різні стани валідації (вільно/зайнято), що впливає на можливість переходу до фінального стану "Пара успішно додана". State Transition ідеально покриває такі життєві цикли об'єкта.

## Аналіз

### Перелік станів (States):
* **S1:** Початковий стан (Розклад відкритий, група не обрана).
* **S2:** Група обрана (Сітка розкладу розблокована для вибору слоту).
* **S3:** Слот обрано (Відкрито модальне вікно вибору дисципліни, викладача та аудиторії).
* **S4:** Аудиторію обрано, очікування валідації (Натиснуто кнопку "Зберегти").
* **S5:** Пара успішно додана (Кінцевий успішний стан).
* **S6:** Стан помилки конфлікту (Відображається повідомлення про зайнятість аудиторії/викладача).

### Таблиця переходів (State-Transition Table):

| Поточний стан | Дія / Умова (Trigger) | Наступний стан | Очікуваний результат (Output) |
| :--- | :--- | :--- | :--- |
| **S1** | Обрати групу з випадаючого списку | **S2** | Колонки розкладу розблоковуються, недоступні дні залишаються заблокованими. |
| **S2** | Клікнути на вільний слот розкладу | **S3** | Відкривається вікно "Додати пару". |
| **S2** | Клікнути на вже зайнятий слот | **S2** | Ігнорування кліку АБО відкриття вікна редагування існуючої пари. |
| **S3** | Натиснути "Скасувати" або хрестик | **S2** | Вікно закривається, повернення до розкладу групи. |
| **S3** | Заповнити дані (вільна аудиторія) та натиснути "Зберегти" | **S4 -> S5** | Пара з'являється у сітці розкладу, вікно закривається. |
| **S3** | Заповнити дані (зайнята аудиторія/викладач) та натиснути "Зберегти" | **S4 -> S6** | З'являється повідомлення про помилку конфлікту (напр. "Аудиторія зайнята"). |
| **S6** | Змінити аудиторію на вільну та натиснути "Зберегти" | **S4 -> S5** | Помилка зникає, пара успішно зберігається. |
