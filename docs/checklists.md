# Чеклісти для тестування

## Модуль: Перегляд розкладу викладача

### Функціональність
- [ ] Перевірка 1: Пошук викладача за повним ПІБ видає коректний результат.
- [ ] Перевірка 2: Пошук викладача за частковим збігом (наприклад, тільки прізвище) працює коректно.
- [ ] Перевірка 3: При виборі викладача автоматично завантажується розклад на поточний робочий тиждень.
- [ ] Перевірка 4: Перемикання на наступний/попередній тиждень оновлює таблицю розкладу відповідно до дат.
- [ ] Перевірка 5: Коректне відображення розкладу для викладача з максимальною завантаженістю (наприклад, 4-5 пар на день).
- [ ] Перевірка 6: Якщо у викладача є "вікна" (наприклад, 1-ша і 3-тя пари), 2-га пара відображається як порожня комірка.
- [ ] Перевірка 7: Перехід за прямим посиланням на розклад конкретного викладача (через ID в URL) відкриває потрібну сторінку.

### Негативні сценарії
- [ ] Перевірка 8: Введення спецсимволів (напр., `@#$%^&*`) або метасимволів у поле пошуку викладача не викликає падіння системи (Error 500).
- [ ] Перевірка 9: Спроба перейти за URL з неіснуючим ID викладача (напр., `/teacher/999999`) відображає коректну сторінку помилки 404.
- [ ] Перевірка 10: Ручна зміна параметрів дати в URL на невалідні (напр., `start_date=2026-15-45`) обробляється коректно без крашу.
- [ ] Перевірка 11: Пошук з використанням SQL/XSS ін'єкцій у полі вводу імені нейтралізується системою.

### UI/UX
- [ ] Перевірка 12: Таблиця розкладу адаптивно стискається та залишається читабельною на мобільних пристроях (ширина 320px - 480px).
- [ ] Перевірка 13: Довгі назви дисциплін або ПІБ викладача переносяться на новий рядок і не ламають структуру таблиці.
- [ ] Перевірка 14: При завантаженні розкладу (зміні тижня) відображається індикатор завантаження (спінер/скелетон).
- [ ] Перевірка 15: У вихідні дні (субота, неділя), якщо пар немає, стовпці або приховуються, або мають чітке повідомлення "Вихідний".
