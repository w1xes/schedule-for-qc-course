# Чеклісти для тестування

## Модуль: Розміщення пари на конкретний час/день у розкладі (адмін-панель)

### Функціональність (Позитивні сценарії)
- [ ] Вибір групи з випадаючого списку коректно розблоковує сітку розкладу для цієї групи.
- [ ] Клік по вільному слоту в розкладі відкриває модальне вікно "Додати пару".
- [ ] У модальному вікні коректно підтягується обраний день та час слоту.
- [ ] Успішне додавання лекції з вибором вільної аудиторії та викладача.
- [ ] Успішне додавання практичного заняття (з можливістю поділу на підгрупи, якщо підтримується).
- [ ] Після збереження, нова пара миттєво відображається у відповідному слоті розкладу без перезавантаження сторінки.
- [ ] Можливість додати пару у найперший слот (напр., 08:00) та найостанніший слот робочого дня.

### Негативні сценарії (Спроби створити конфлікти)
- [ ] Спроба зберегти пару без вибору дисципліни/викладача/аудиторії (перевірка валідації обов'язкових полів).
- [ ] Спроба призначити викладача на слот, в якому він вже веде пару в іншої групи. (Очікується помилка конфлікту).
- [ ] Спроба призначити пару в аудиторію, яка вже зайнята іншою групою в цей самий час. (Очікується блокування або попередження).
- [ ] Спроба додати дві різні пари для однієї і тієї ж групи на один і той самий часовий слот.
- [ ] Спроба додати пару на вихідний день (неділя), якщо це не передбачено бізнес-логікою.
- [ ] Спроба зберегти пару з аудиторією, місткість якої менша за кількість студентів у групі (опціонально, якщо є такий функціонал).

### UI/UX
- [ ] Заблоковані колонки (наприклад, минулі дні або слоти іншої зміни) візуально відрізняються (сірий фон, іконка замка).
- [ ] Курсор змінюється на `not-allowed` при наведенні на заблокований слот.
- [ ] При виникненні конфлікту розкладу повідомлення про помилку підсвічується червоним кольором і чітко пояснює причину (напр., "Аудиторія 305 вже зайнята").
- [ ] Елементи модального вікна не виходять за межі екрану на менших роздільних здатностях (напр., на планшеті).
- [ ] Кнопка "Зберегти" стає неактивною (`disabled`) до моменту заповнення всіх обов'язкових полів.
- [ ] Довгі назви предметів коректно переносяться на новий рядок у картці заняття в сітці розкладу, не ламаючи верстку.
